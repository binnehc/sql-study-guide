WINDOW FUNCTIONS

Window functions calculate across a set of rows without collapsing them. Unlike GROUP BY which gives one row per group, window functions add a new column to every existing row.

Core Syntax

function() OVER ( PARTITION BY col ORDER BY col )
PARTITION BY = reset the calculation for each group  ·  ORDER BY = define the sequence within each partition.

Think of PARTITION BY department as splitting the table into separate groups — the window function runs independently inside each one:

dept = 'Engineering'
Alice$120krank 1
Bob$110krank 2
Eve$110krank 2 (tie)
Carol$80krank 4
dept = 'Marketing'
Dave$90krank 1 ← resets
Grace$72krank 2
dept = 'Finance'
Frank$95krank 1 ← resets

Each partition gets its own independent ranking — no mixing between groups

ROW_NUMBER vs RANK vs DENSE_RANK

All three assign a number to each row within a group. The difference appears when there are ties (equal values).

ROW_NUMBER()

Always unique. Ties get arbitrary order. No gaps. Use when you need exactly one row per group.

RANK()

Ties get the same rank. The next rank skips. E.g., two 2nds → next is 4th. Creates gaps.

DENSE_RANK()

Ties get the same rank. The next rank does NOT skip. E.g., two 2nds → next is 3rd. No gaps.

SQL · All Three Ranking Functions
SELECT
    name,
    department,
    salary,
    ROW_NUMBER()  OVER (PARTITION BY department ORDER BY salary DESC) AS row_num,
    RANK()        OVER (PARTITION BY department ORDER BY salary DESC) AS rnk,
    DENSE_RANK()  OVER (PARTITION BY department ORDER BY salary DESC) AS dense_rnk
FROM employees;
Other Useful Window Functions
SUM() OVER()

Running total or group total. Add ORDER BY to make it cumulative.

LAG() / LEAD()

Access a value from the previous or next row. Great for "change from last month" patterns.

NTILE(n)

Divide rows into n equal buckets. NTILE(4) creates quartiles.

FIRST_VALUE() / LAST_VALUE()

Get the first or last value in the window. Useful for comparing each row against the group leader.

LAG looks back one row · LEAD looks forward one row · rows at the boundary return NULL:

monthrevenueLAG(revenue)LEAD(revenue)
Jan$50k NULL ←boundary $60k ↓
Feb$60k $50k ↑ $45k ↓
Mar$45k $60k ↑ $80k ↓
Apr$80k $45k ↑ NULL →boundary

Use revenue - LAG(revenue) OVER (ORDER BY month) to compute month-over-month change

SQL · Running Total + LAG Example
-- Running salary total per department (ordered by hire date)
SELECT
    name, department, salary,
    SUM(salary) OVER (
        PARTITION BY department
        ORDER BY     hire_date
    ) AS running_total
FROM employees;

-- Salary change vs previous employee hired in same dept
SELECT
    name, department, salary,
    LAG(salary) OVER (
        PARTITION BY department
        ORDER BY     hire_date
    ) AS prev_salary
FROM employees;
The "Top N per Group" Pattern — one of the most common real-world queries

"Return the top-earning employee per department" — this pattern appears constantly. The trick: ROW_NUMBER in a CTE, then filter in the outer query.

SQL · Top 1 per Department
WITH ranked AS (
    SELECT
        name, department, salary,
        ROW_NUMBER() OVER (
            PARTITION BY department
            ORDER BY     salary DESC
        ) AS rn
    FROM employees
)
SELECT name, department, salary
FROM   ranked
WHERE  rn = 1;   -- change to rn <= 3 for "top 3 per group"

ROW_NUMBER vs DENSE_RANK here

If ties exist and you want all tied employees at rank 1, replace ROW_NUMBER() with DENSE_RANK() in the CTE (alias it as dr to avoid confusion), then filter WHERE dr <= 1. If you want exactly one row per group regardless of ties, keep ROW_NUMBER(). The choice depends on the problem.

CASE WHEN — SQL's if/else

CASE WHEN evaluates conditions and returns a value. It can appear in SELECT (to label/transform), inside aggregates (to count/sum conditionally), or in ORDER BY.

SQL · CASE WHEN Patterns
-- 1. Label/categorize rows
SELECT name, salary,
    CASE
        WHEN salary >= 100000 THEN 'Senior'
        WHEN salary >= 75000  THEN 'Mid'
        ELSE                        'Junior'
    END AS level
FROM employees;

-- 2. Conditional count (count only rows that match)
SELECT
    COUNT(CASE WHEN salary >= 100000 THEN 1 END) AS senior_count,
    COUNT(CASE WHEN salary  < 100000 THEN 1 END) AS junior_count
FROM employees;

Practice

Write a query to rank employees by salary within each department (highest = rank 1) using DENSE_RANK. Show name, department, salary, and rank.

Table: employees (employee_id, name, department, salary) — use DENSE_RANK so ties don't create gaps

SQL
Answer
SELECT
    name,
    department,
    salary,
    DENSE_RANK() OVER (
        PARTITION BY department
        ORDER BY     salary DESC
    ) AS salary_rank
FROM employees;
