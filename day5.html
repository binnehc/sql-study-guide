A CTE is a named temporary result set defined at the top of your query. It only exists for the duration of that query. The main benefit: breaking complex logic into readable, named steps instead of deeply nested subqueries.

Without CTE — hard to read
SQL
SELECT department
FROM (
  SELECT department,
         SUM(salary) AS total
  FROM   employees
  GROUP BY department
) dept_totals
WHERE total > 200000;
With CTE — clean and named
SQL
WITH dept_totals AS (
  SELECT department,
         SUM(salary) AS total
  FROM   employees
  GROUP BY department
)
SELECT department
FROM   dept_totals
WHERE  total > 200000;

Reading the WITH syntax

WITH dept_totals AS ( SELECT department, SUM(salary) AS total FROM employees GROUP BY department )
SELECT department FROM dept_totals WHERE total > 200000

  • dept_totals — the name you give the CTE. Use anything descriptive. This is what you reference later.
  • inner query — any valid SELECT. This is your intermediate step — write it exactly as you would a normal query.
  • final SELECT — your actual query. Reads from the CTE name like it's a real table. Only this part produces visible output.

CTEs are essential for window functions. You cannot filter on a window function result in the same query — you must wrap it in a CTE first, then filter in the outer query.

SQL · Why CTEs Enable Window Functions
-- This FAILS — you can't use ROW_NUMBER alias in WHERE of same query
SELECT *, ROW_NUMBER() OVER (...) AS rn
FROM   employees
WHERE  rn = 1;          -- ❌ ERROR: rn doesn't exist yet

-- This WORKS — CTE saves the result, outer query filters it
WITH ranked AS (
    SELECT *, ROW_NUMBER() OVER (
        PARTITION BY department
        ORDER BY     salary DESC
    ) AS rn
    FROM employees
)
SELECT name, department, salary
FROM   ranked
WHERE  rn = 1;          -- ✓ top earner per department

You can chain multiple CTEs with a comma — each can reference the ones above it:

SQL · Chained CTEs
WITH

step_one AS (
    SELECT ... FROM raw_table
),

step_two AS (
    SELECT ... FROM step_one   -- references the first CTE
    WHERE  ...
)

SELECT * FROM step_two;

Practice

Rewrite this using a CTE: find departments where total salary exceeds 200,000.

Table: employees (employee_id, name, department, salary) — replace the nested subquery with a named CTE

SQL
Answer
WITH dept_totals AS (
    SELECT   department,
             SUM(salary) AS total
    FROM     employees
    GROUP BY department
)
SELECT department
FROM   dept_totals
WHERE  total > 200000;
