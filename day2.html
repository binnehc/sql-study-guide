Day 02 · Combining Tables

JOIN TYPES

A JOIN combines rows from two tables based on a matching column — usually a shared ID. The type of JOIN controls what happens to rows that don't match.

INNER JOIN
Matching rows only
LEFT JOIN
All left + matched right
RIGHT JOIN
All right + matched left
FULL OUTER JOIN
All rows from both tables

Table Aliases

In JOIN queries you'll see FROM employees e — the e is a table alias, a shorthand nickname. It lets you write e.name instead of employees.name. Aliases are required when you join a table to itself, and just cleaner everywhere else.

INNER JOIN — only matching rows from both tables

The most common join. Returns a row only when the key exists in both tables. Dave (NULL dept) and Finance (no employees) are both excluded.

SQL
SELECT e.name, d.dept_name
FROM       employees   e
INNER JOIN departments d ON e.dept_id = d.dept_id;
namedept_name
AliceEngineering
BobEngineering
CarolMarketing

Dave (NULL dept) → excluded. Finance dept (no employees) → excluded.

LEFT JOIN — all rows from the left table, matched or not

Returns every row from the left table (employees). If there's no match in the right table, the right-side columns are NULL. Use this to find "things without a match" — e.g., employees with no department.

SQL
SELECT e.name, d.dept_name
FROM      employees   e
LEFT JOIN departments d ON e.dept_id = d.dept_id;

-- Find employees with NO department:
SELECT e.name
FROM      employees   e
LEFT JOIN departments d ON e.dept_id = d.dept_id
WHERE     d.dept_id IS NULL;   -- the "no match" trick
namedept_name
AliceEngineering
BobEngineering
CarolMarketing
DaveNULL ← no dept

RIGHT JOIN — all rows from the right table, matched or not

Mirror of LEFT JOIN. Returns every row from the right table (departments). Finance shows up even though no employees belong to it. Rarely used — most developers just swap the table order and use LEFT JOIN instead.

SQL
SELECT e.name, d.dept_name
FROM       employees   e
RIGHT JOIN departments d ON e.dept_id = d.dept_id;
namedept_name
AliceEngineering
BobEngineering
CarolMarketing
NULLFinance ← no employees

FULL OUTER JOIN — all rows from both tables

The union of LEFT and RIGHT. Returns every row from both tables. Non-matching sides get NULL. Use when you need a complete picture across two tables regardless of matches.

SQL
SELECT e.name, d.dept_name
FROM            employees   e
FULL OUTER JOIN departments d ON e.dept_id = d.dept_id;
namedept_name
AliceEngineering
BobEngineering
CarolMarketing
DaveNULL
NULLFinance

The "Missing Data" Pattern

LEFT JOIN + WHERE right.id IS NULL is one of the most useful patterns in real queries. It finds rows in the left table with no match on the right — e.g., "customers who never placed an order", "users with no activity."

  • I can find "missing data" with LEFT JOIN + WHERE right.id IS NULL

Practice

Write a query to find all employees who have no department assigned. Return only the employee name.

Tables: employees (id, name, dept_id) · departments (dept_id, dept_name)

SQL
Answer
SELECT    e.name
FROM      employees   e
LEFT JOIN departments d ON e.dept_id = d.dept_id
WHERE     d.dept_id IS NULL;
INNER JOIN

Only rows with a match in both tables. Most common.

LEFT JOIN

All rows from left table; NULL on right if no match.

RIGHT JOIN

All rows from right table; NULL on left if no match.

FULL OUTER JOIN

All rows from both; NULL where no match on either side.

-- INNER: only matches
FROM orders o JOIN customers c ON o.cust_id = c.id

-- LEFT: keep all orders even without a customer
FROM orders o LEFT JOIN customers c ON o.cust_id = c.id
FROM a JOIN b ON ...

Join condition after ON. Use table aliases to keep it short.

Chaining JOINs

Stack multiple JOINs — each adds another table to the row.

Column ambiguity

If two tables share a column name, prefix: e.id vs d.id.

FROM employees e
JOIN depts d       ON e.dept_id = d.id
JOIN locations loc ON d.loc_id  = loc.id;
Find rows with no match

LEFT JOIN + WHERE right.col IS NULL = rows that don't appear in the right table.

Classic pattern

Users who never ordered, riders with no trips, items never sold.

SELECT r.id, r.name
FROM   riders r
LEFT JOIN trips t ON r.id = t.rider_id
WHERE  t.rider_id IS NULL;
